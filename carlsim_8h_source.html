454  /*
455  * \brief Sets the integration method for the simulation
456  *
457  * This function specifies the integration method for the simulation. Currently, the chosen integration method
458  * will apply to all neurons in the network.
459  *
460  * The basic simulation time step is 1ms, meaning that spike times cannot be retrieved with sub-millisecond
461  * precision. However, the integration time step can be lower than 1ms, which is specified by numStepsPerMs.
462  * A numStepsPerMs set to 10 would take 10 integration steps per 1ms simulation time step.
463  *
464  * By default, the simulation will use Forward-Euler with an integration step of 0.5ms (i.e.,
465  * <tt>numStepsPerMs</tt>=2).
466  *
467  * Currently CARLsim supports the following integration methods:
468  * - FORWARD_EULER: The most basic, forward-Euler method. Suggested value for <tt>numStepsPerMs</tt>: >= 2.
469  * - RUNGE_KUTTA4: Fourth-order Runge-Kutta (aka classical Runge-Kutta, aka RK4).
470  * Suggested value for <tt>numStepsPerMs</tt>: >= 10.
471  *
472  * \STATE ::CONFIG_STATE
473  * \param[in] method the integration method to use
474  * \param[in] numStepsPerMs the number of integration steps per 1ms simulation time step
475  *
476  * \note Note that the higher numStepsPerMs the slower the simulation may be, due to increased computational load.
477  * \since v3.1
478  */
479  void setIntegrationMethod(integrationMethod_t method, int numStepsPerMs);
